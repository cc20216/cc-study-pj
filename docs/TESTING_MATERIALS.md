```python
def test_login_success(self):
    """测试登录成功"""
    response = self.client.post('/login/', {
        'username': 'testuser',
        'password': 'testpassword123'
    })
    self.assertRedirects(response, '/')
    self.assertTrue('_auth_user_id' in self.client.session)

def test_login_failure(self):
    """测试登录失败"""
    response = self.client.post('/login/', {
        'username': 'testuser',
        'password': 'wrongpassword'
    })
    self.assertContains(response, '用户名或密码错误')
    self.assertFalse('_auth_user_id' in self.client.session)
```

#### 1.3.2 进度提交测试
```python
def test_submit_daily_progress(self):
    """测试提交每日进度"""
    # 创建测试模板和模块
    template = StudyTemplate.objects.create(name='测试模板', creator=self.user)
    module = TemplateModule.objects.create(template=template, name='测试模块', target_value=100)
    
    # 提交进度
    response = self.client.post('/progress/submit/', {
        'template': template.id,
        'module_{}'.format(module.id): 50,
        'learning_time': 1.5
    })
    
    # 验证重定向
    self.assertRedirects(response, '/progress/dashboard/')
    
    # 验证数据已保存
    progress = DailyProgress.objects.filter(user=self.user, module=module).first()
    self.assertIsNotNone(progress)
    self.assertEqual(progress.actual_value, 50)
    
    # 验证学习会话已创建
    session = StudySession.objects.filter(user=self.user, subject=template.name).first()
    self.assertIsNotNone(session)
    self.assertEqual(session.duration_minutes, 90)  # 1.5小时 = 90分钟
```

## 3. 故障恢复方案

### 3.1 系统崩溃恢复流程

#### 3.1.1 故障检测
- **监控系统**：使用监控工具（如Prometheus）监控系统状态
- **日志分析**：定期分析系统日志，及时发现异常
- **健康检查**：实现API健康检查端点

#### 3.1.2 故障分类
| 故障类型 | 影响范围 | 恢复时间 |
|---------|---------|----------|
| 应用崩溃 | 全部用户 | 5-10分钟 |
| 数据库故障 | 全部用户 | 10-30分钟 |
| 网络故障 | 部分用户 | 5-15分钟 |
| 硬件故障 | 全部用户 | 30-60分钟 |

#### 3.1.3 恢复步骤

**应用崩溃恢复**：
1. 停止当前进程
2. 检查错误日志
3. 修复代码问题（如果需要）
4. 重启应用服务器
5. 验证系统正常运行

**数据库故障恢复**：
1. 停止应用服务
2. 检查数据库状态
3. 从最近的备份恢复数据库
4. 应用未备份的事务日志
5. 重启数据库服务
6. 重启应用服务
7. 验证数据完整性

**网络故障恢复**：
1. 检查网络连接
2. 重启网络设备（如果需要）
3. 验证网络连接恢复
4. 检查系统服务是否正常

**硬件故障恢复**：
1. 切换到备用服务器
2. 从备份恢复数据
3. 配置网络和服务
4. 验证系统正常运行
5. 修复或更换故障硬件
